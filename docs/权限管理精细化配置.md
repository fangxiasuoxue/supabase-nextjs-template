# 权限管理精细化配置

## 目前的问题
- 对用户授权时，不能同时选read、wirte、manage，只能选中一个，再选其它的会报错
- 以ip管理为例，授权read后，所有的IP全部会出现，而且每一条IP后面，都有分配按钮，这样一个普通用户，就可以再将IP分配给其它人，这样是不对的

## 权限设计应该如下设计
- 权限总共分：菜单权限menu、数据权限（read、wirte、manage）四种类型
- 菜单权限决定用户是否能看到左侧菜单，与直接访问路由，不与读写管数据权限冲突
- 数据权限是对数据进行的操作
### 菜单权限
- 对左侧是否出现菜单进行控制
### 数据权限，分读、写、管理
#### 读（read）
- 对所有数据有只读权限，但不包括写和管理权限
#### 写（Write）
- 对数据有新增权限，对自己是owner的数据，可以删除，但不是真删除，只修改删除标识位
- 对已有的非owner数据有修改权限，但无删除权限
#### 管理（manage）
- 管理数据行为一般有：删除非自己owner的数据，分配、同步这类的操作
- 分配区别与读权限，是分配是对部分指定的数据进行读的授权，分配后的数据，不授权read、只授权写或管，就可以对部分数据进行写或管的操作
